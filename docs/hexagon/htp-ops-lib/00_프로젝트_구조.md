# htp-ops-lib 프로젝트 구조 분석

> **분석 대상**: [htp-ops-lib/](../../../htp-ops-lib/) 폴더 전체  
> **프로젝트 설명**: Qualcomm Hexagon NPU(cDSP)에서 LLM 추론을 위한 커스텀 연산자 라이브러리

## 0. 프로젝트 개요

이 라이브러리는 논문 [Scaling LLM Test-Time Compute with Mobile NPU on Smartphones](https://arxiv.org/abs/2509.23324)의 구현체로, Qualcomm Snapdragon 8 Gen 2 이상의 SoC에서 **FP16 HMX(Hexagon Matrix eXtension)**를 활용한 LLM 추론 가속을 수행한다.

### 하드웨어 요구사항
- Qualcomm Snapdragon 8 Gen 2+ (Hexagon DSP v73+)
- **FP16 HMX** 지원 필수 (일부 중저가 기기에서는 미지원)

### 소프트웨어 요구사항
- Hexagon SDK 6.x (검증 버전: 6.0.0.2)

## 3. 주요 모듈별 역할

### 3.1 통신 계층 (Communication Layer)

| 모듈 | 파일 위치 | 역할 |
|------|-----------|------|
| **FastRPC IDL** | [include/htp_ops.idl](../../../htp-ops-lib/include/htp_ops.idl) | `init_backend`, `create_channel`, `rms_norm_f32`, `mat_mul_permuted_w16a32`, `test_ops` 등의 RPC 인터페이스 정의 |
| **메시지 프로토콜** | [include/message.h](../../../htp-ops-lib/include/message.h) | Host-DSP 간 공유 메모리 기반 저지연 메시지 채널. `MessageHeader` → `RequestHeader` → `OpComputeRequest` 구조 |
| **메시지 수신 루프** | [src/dsp/commu.c](../../../htp-ops-lib/src/dsp/commu.c) | DSP 측 폴링 기반 메시지 수신자. `memd_aq` (acquire) / `memd_rl` (release) 원자적 메모리 접근으로 동기화 |
| **세션 관리** | [src/host/session.c](../../../htp-ops-lib/src/host/session.c) | DSP 세션 열기/닫기, unsigned PD 설정, FastRPC QoS 모드 활성화 (목표 지연 50μs) |

### 3.2 하드웨어 자원 관리

| 모듈 | 파일 위치 | 역할 |
|------|-----------|------|
| **전력 관리** | [src/dsp/power.c](../../../htp-ops-lib/src/dsp/power.c) | DCVS (Dynamic Clock and Voltage Scaling) 설정 → TURBO_L3 모드, HMX 전원 ON/OFF |
| **HMX 관리자** | [src/dsp/hmx_mgr.c](../../../htp-ops-lib/src/dsp/hmx_mgr.c) | `HAP_compute_res` API로 HMX 자원 획득, 스레드별 HMX 실행 활성화, 스핀락 기반 HMX 유닛 독점 제어 |
| **VTCM 관리자** | [src/dsp/vtcm_mgr.cc](../../../htp-ops-lib/src/dsp/vtcm_mgr.cc) | VTCM(Vector Tightly-Coupled Memory) 전체 할당, 명명된 영역 예약 (예: exp2 테이블) |
| **워커 풀** | [src/dsp/worker_pool.c](../../../htp-ops-lib/src/dsp/worker_pool.c) | QuRT 기반 멀티스레드 워커 풀, 최대 6개 HVX 스레드 지원, HMX 접근 허용 옵션 |
| **메모리 매핑** | [src/dsp/mmap_mgr.cc](../../../htp-ops-lib/src/dsp/mmap_mgr.cc) | `HAP_mmap_get/put`으로 rpcmem fd → DSP 주소 매핑 관리 |

### 3.3 핵심 연산자 (Core Operators)

| 연산자 | 파일 위치 | 사용 하드웨어 | 역할 |
|--------|-----------|--------------|------|
| **MatMul (FP16 Weight)** | [src/dsp/ops/mat_mul.c](../../../htp-ops-lib/src/dsp/ops/mat_mul.c) | HMX + HVX + DMA | FP32 activation × FP16 weight → FP32 output |
| **MatMul (양자화 Weight)** | [src/dsp/ops/mat_mul.c](../../../htp-ops-lib/src/dsp/ops/mat_mul.c) | HMX + HVX + DMA | FP32 activation × Q4_0/Q8_0/IQ4_NL weight → FP32 output (실시간 역양자화 포함) |
| **FlashAttention** | [src/dsp/ops/flash_attn.c](../../../htp-ops-lib/src/dsp/ops/flash_attn.c) | HMX + HVX | QK^T 내적(HMX), Softmax(HVX), P·V 누적(HMX) |
| **FlashAttention (비정렬 D)** | [src/dsp/ops/flash_attn_sp_hdim.c](../../../htp-ops-lib/src/dsp/ops/flash_attn_sp_hdim.c) | HMX + HVX | head_dim이 64의 배수가 아닌 경우 처리 |
| **RMS Norm** | [src/dsp/ops/rms_norm.c](../../../htp-ops-lib/src/dsp/ops/rms_norm.c) | HVX | FP32 RMS Normalization |
| **Precompute Table** | [src/dsp/ops/precompute_table.c](../../../htp-ops-lib/src/dsp/ops/precompute_table.c) | HVX | VTCM 내 exp2 룩업 테이블 사전 계산 (safe softmax용) |

### 3.4 연산 디스패치

| 모듈 | 파일 위치 | 역할 |
|------|-----------|------|
| **연산자 레지스트리** | [include/op_reg.h](../../../htp-ops-lib/include/op_reg.h) | `HtpOpsIndex` enum: 6종 연산 ID + 파라미터 구조체 (`MatMulParams`, `FlashAttnParams` 등) |
| **실행기** | [src/dsp/op_executor.cc](../../../htp-ops-lib/src/dsp/op_executor.cc) | `OpComputeRequest.op`에 따라 해당 연산자를 호출, D-cache invalidate/flush 처리 |

## 4. 데이터 흐름 개요

```
┌─────────────────────────────────────────────────────────┐
│                    Host (Android CPU)                    │
│  ┌──────────┐   ┌─────────────┐   ┌──────────────────┐ │
│  │ session.c │──▶│ FastRPC/IDL │──▶│ Shared Memory   │ │
│  │ op_export │   │ (htp_ops)   │   │ (rpcmem)        │ │
│  └──────────┘   └─────────────┘   └────────┬─────────┘ │
│                                             │           │
└─────────────────────────────────────────────┼───────────┘
                                              │ fd 전달
┌─────────────────────────────────────────────┼───────────┐
│                   DSP (cDSP/Hexagon NPU)    │           │
│  ┌──────────┐   ┌──────────────┐   ┌───────▼─────────┐ │
│  │ commu.c  │──▶│ op_executor  │──▶│ DDR Memory      │ │
│  │ (수신 루프)│   │ (디스패치)    │   │ (버퍼:in/out)  │ │
│  └──────────┘   └──────┬───────┘   └─────────────────┘ │
│                        │                                │
│                 ┌──────▼───────┐                        │
│                 │  mat_mul.c   │   ┌──────────────────┐ │
│                 │ flash_attn.c │──▶│ VTCM (8MB)      │ │
│                 │ rms_norm.c   │   │ ┌──────────────┐ │ │
│                 └──────────────┘   │ │ HMX Tiles    │ │ │
│                                    │ │ (Crouton Fmt)│ │ │
│                                    │ └──────────────┘ │ │
│                                    └──────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 통신 방식 2가지

1. **FastRPC 방식**: `htp_ops.idl`에 정의된 함수를 직접 RPC 호출 → 단순하지만 호출 오버헤드 존재
2. **메시지 채널 방식**: 공유 메모리 버퍼를 통한 폴링 기반 통신 → 저지연 (acquire/release 시맨틱 사용)

## 5. 지원하는 연산 목록

> 코드 위치: [include/op_reg.h](../../../htp-ops-lib/include/op_reg.h)

| Op Index | 연산명 | 설명 |
|----------|--------|------|
| `HTP_OPS_RMS_NORM_F32` | RMS Norm | FP32 입출력, HVX 사용 |
| `HTP_OPS_MAT_MUL_PERMUTED_W16A32` | MatMul (FP16 Weight) | FP32 activation × pre-permuted FP16 weight |
| `HTP_OPS_MAT_MUL_PERMUTED_W4D16A32` | MatMul (Q4_0) | Q4_0 weight → 실시간 FP16 역양자화 → HMX matmul |
| `HTP_OPS_MAT_MUL_PERMUTED_W8D16A32` | MatMul (Q8_0) | Q8_0 weight → 실시간 FP16 역양자화 → HMX matmul |
| `HTP_OPS_MAT_MUL_PERMUTED_W4D16A32_IQ4_NL` | MatMul (IQ4_NL) | IQ4_NL weight (비선형 양자화) → HMX matmul |
| `HTP_OPS_FLASH_ATTN_QO_F32_KV_F16` | FlashAttention | Q/O: FP32, K/V: FP16, GQA 지원 |
